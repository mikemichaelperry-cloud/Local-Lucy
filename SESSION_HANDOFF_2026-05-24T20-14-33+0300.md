## Executive Summary

Fixed **two related bugs** in the social-greeting handling path:

1. **Routing bug:** Queries like `"How are you today Lucy?"` and `"What's up?"` were misrouting to `NEWS`/`AUGMENTED` instead of `LOCAL`.
2. **Answer-generation bug:** Even when routed correctly, the local model was confused by temporal keywords (e.g. "evening") and returned `"This requires evidence mode"` + full KEEL regurgitation + echoing the user's question.

Both are fixed and tested. **Zero regressions** in the synthetic adversarial suite (1099 passed, 514 skipped — identical to baseline). Also fixed **16 pre-existing broken tests** that were failing due to stale expectations from the Phase 2C `write_batch` API change.

## Bug 1 — Routing: Social Greetings Misrouted to NEWS/AUGMENTED

### Symptom
`"How are you today Lucy?"` → `NEWS` → "Unable to fetch live news..."

### Root Cause (two paths)

**Path A — long greetings:** The BERT classifier saw "today" as a temporal signal and put `NEWS` in `candidate_routes`. The embedding router correctly predicted `LOCAL` with 99% confidence, but `embedding_local_override` logic trusts the classifier's `candidate_routes` over the embedding router when `NEWS` is present.

**Path B — short greetings:** `"What's up?"` (10 chars) hit the short-query guard (`len < 12`). The short-query guard checks the feedback buffer and inherits the prior route. The feedback buffer had a prior `NEWS` exchange, so the greeting was dragged to `AUGMENTED`.

### Fix

Added `_is_social_greeting()` guard in `tools/router_py/classify.py` with 18 compiled regex patterns covering:
- `How are you (today/doing/feeling)?`
- `What's up?`, `How's it going?`, `Good morning/evening`
- `Hey Lucy`, `Hi Lucy`, `Hello Lucy`, etc.
- Carefully excludes non-greetings like `"How are you going to fix this?"`

Two override points:
1. **Embedding-local path** — forces `LOCAL` even when classifier includes `EVIDENCE/NEWS` in `candidate_routes`
2. **Short-query path** — returns `LOCAL` immediately without checking feedback buffer

## Bug 2 — Answer Generation: KEEL Regurgitation on Greetings

### Symptom
`"Good evening Lucy, how are you this evening?"` →
```
Path: Local answer
Route confidence: 0.86
This requires evidence mode. Run: run online:
[KEEL — Hard Constraints] ... [END KEEL]
Good evening Lucy, how are you this evening?
```

### Root Cause
Even with correct `LOCAL` routing, the model was confused. The prompt instruction says *"For live data... I say 'Requires evidence mode.'"*. The model interpreted "evening" as a time query, output the refusal, then regurgitated the entire KEEL block and echoed the user's question.

### Fix

Added `_is_social_greeting()` and `_greeting_response()` in `tools/router_py/local_answer.py`. Social greetings now **bypass the model entirely** — no prompt assembly, no Ollama call, no chance of KEEL regurgitation or time-sensitive false positives.

Returns warm, context-appropriate responses:
- `"Good morning!"` → "Good morning! I'm doing well..."
- `"Good evening!"` → "Good evening! I'm doing well..."
- `"What's up?"` → "Not much! Just here and ready to help..."
