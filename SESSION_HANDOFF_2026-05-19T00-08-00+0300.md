## What Was Done This Session

### 1. Critical Regression Fix: Missing `_is_historical_query()`
**File:** `tools/router_py/classify.py`

Commit `bc93604` added a call to `_is_historical_query()` inside `select_route()` but never defined the function. The `NameError` was swallowed by `except Exception: pass`, causing **every query reaching the embedding router to silently fall back to LOCAL**.

Impact:
- Medical queries → LOCAL (should be EVIDENCE)
- Veterinary queries → LOCAL (should be EVIDENCE)
- Financial live data → LOCAL (should be AUGMENTED)
- News/synthesis → LOCAL (should be NEWS/AUGMENTED)

Fix: Defined the function with word-boundary matching for short markers ("era" → must not match inside "consideration"). Also removed `"who won"` / `"who lost"` which matched current sports results.

### 2. Silent Exception Catch Fixed
**File:** `tools/router_py/classify.py`

The bare `except Exception: pass` around the embedding router path has been replaced with explicit logging:
- Decision log now contains `guards_fired: ["router_failure", "exception_NameError"]`
- stderr prints: `[ROUTER_EXCEPTION_FALLBACK_LOCAL] NameError: name '_is_xxx' is not defined`
- Full traceback printed to stderr

**This prevents future missing-function bugs from wearing camouflage.**

## Architecture State

### Router Decision Flow
```
select_route()
├── force_local? → LOCAL
├── hostile_override? → LOCAL
├── short_query (< 12 chars)? → LOCAL
├── garbage/noise? → LOCAL
├── financial_ephemeral? → AUGMENTED (early return)
├── embedding_router.predict()
│   └── Returns route, confidence, guards_fired
├── medical/vet safety override → EVIDENCE
├── weather keyword override → WEATHER
├── clear-news override → NEWS
├── historical context override → LOCAL
├── weather-pattern negation → LOCAL
├── capability query override → LOCAL
├── personal-finance reasoning override → LOCAL
├── embedding_local_override
│   ├── medical/vet → EVIDENCE
│   ├── NEWS + synthesis → AUGMENTED
│   ├── NEWS → NEWS
│   └── else → AUGMENTED
├── route == NEWS + synthesis → AUGMENTED
├── route == TIME → TIME
├── route == WEATHER (+has_weather_keywords) → WEATHER
├── route == AUGMENTED/EVIDENCE (+medical/vet) → EVIDENCE
└── memory follow-up override → LOCAL
```

### Key Principle
**Deterministic guards for hard boundaries, embeddings for fuzzy semantic classification.** Medical, legal, financial-live, conflict-live, hostile, memory/HMI remain guard-dominated. Embedding only decides when no guard fires.
