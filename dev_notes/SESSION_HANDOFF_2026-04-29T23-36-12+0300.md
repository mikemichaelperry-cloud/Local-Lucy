## 2. Phase 2: Auto-Summarization + Archived History ✅

### New tables (auto-created on first use)
- `session_summaries` — LLM-generated session summaries
- `summary_embeddings` — 4096-dim Ollama embeddings for semantic search
- `archived_turns` — Full conversation history preserved after summarization
- `session_metadata` — Auto-naming (first user query becomes session name)

### Trigger logic: `maybe_summarize_session(session_id, threshold=20)`
- Fires when active turn count ≥ threshold
- Calls Ollama `/generate` to produce a concise summary
- Calls Ollama `/embeddings` to vectorize the summary
- Archives all active turns, clears active table

### Context assembly: `_assemble_context()`
```
[Related/Previous session summaries]   ← cross-session (Augmented only)
[Current session summary]              ← always included in deep mode
[Recent turns (last 4)]                ← always included
```

## 3. Phase 3: Semantic Cross-Session Recall ✅

### Architecture decision: Mode-gated recall
| Mode | Cross-session recall? | Reason |
|------|----------------------|--------|
| **Local** | ❌ No | 8B model easily confused by mixed contexts |
| **Augmented** | ✅ Yes | Higher capacity path can handle richer context |
| **Auto (shallow)** | ❌ No | Standalone queries don't need history |
| **Auto (deep)** | ✅ Yes (Augmented path) | Follow-ups get full semantic context |

### Semantic search: `find_relevant_sessions(query)`
- Embeds query via Ollama `/embeddings`
- Cosine similarity against all `summary_embeddings`
- Conservative thresholds (see Environment Variables below)

### Auto depth detection: `_detect_context_depth(query)`
Regex-based heuristics classify queries as **shallow** or **deep**:
- **Shallow**: pronouns ("it", "he", "that"), follow-up phrases ("what about", "tell me more"), short queries (< 5 words)
- **Deep**: standalone queries that benefit from broader context

### Prompt structure: `local_answer.py` `_build_prompt()`
```
{instruction}

Tone: Warm, calm, clear...
{budget_instruction}

{conversation_block}
{memory_block}
---

{context_block}User: {query}
```
The explicit `---` separator prevents the 8B model from confusing memory context with user instructions.

### Tests: `tools/tests/test_memory_semantic.py` + `test_memory_archive.py` + `test_memory_auto_depth.py`
- 12 semantic tests (embedding, similarity ranking, top-k, gap filtering)
- 8 archive tests (full history preservation, retrieval, session isolation)
- 10 auto-depth tests (pronoun detection, follow-up phrases, short queries)

## 4. Safety Gates & Conservative Thresholds

### Environment Variables (all optional, defaults are conservative)
```bash
# Core toggle
export LUCY_SESSION_MEMORY=1          # Enable memory layer (default: 0)

# Semantic search thresholds
export LUCY_MEMORY_SIMILARITY_THRESHOLD=0.70    # Min cosine similarity (default: 0.70)
export LUCY_MEMORY_REQUIRE_TOP_GAP=0.10         # Min gap between best and second-best (default: 0.10)
export LUCY_MEMORY_MAX_INJECTED_SESSIONS=1      # Max cross-session summaries (default: 1)

# Database path
export LUCY_MEMORY_DB_PATH=/path/to/memory.db   # Default: tools/memory/memory.db
```

### Why these thresholds?
Real embedding tests with `local-lucy` 8B revealed:
- "What transformer should I buy?" scored **0.670** vs tubes session — leakage risk at thresholds < 0.60
- "Would you consider him democratically elected?" matched **Charlemagne** session (0.622) because embeddings conflate **topics**, not **entities**

Cross-session semantic recall is therefore **disabled in Local mode entirely**, and gated behind conservative thresholds in Augmented mode.
