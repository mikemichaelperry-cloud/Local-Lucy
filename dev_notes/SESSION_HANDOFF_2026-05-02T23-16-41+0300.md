## 1. NEWS Path v2 — Architecture Changes

### What Changed

| Aspect | Before | After |
|--------|--------|-------|
| Fetch strategy | Sequential `urllib` (14 feeds × ~2s = ~28s worst case) | Parallel `aiohttp` with `asyncio.gather()` (~2–6s) |
| Caching | None | 5-minute in-memory cache → **None** (removed entirely) |
| History | Persisted to `request_history.jsonl` | **Not persisted** (ephemeral display only) |
| LLM involvement | None (correct) | None (unchanged) |
| Error handling | All-or-nothing | Partial results accepted (some feeds may fail) |
| Description length | Full RSS description (often 300+ chars) | Truncated to ~200 chars at sentence boundary |
| Voice format | Headlines only, no description | Headline + source + brief description |
| HTML format | Existed but unused | Kept for future use; display uses plain text + auto-link |

### Files Modified

- **`tools/router_py/news_provider.py`** (749 lines) — Full rewrite:
  - `fetch_world_news_async()` — parallel async fetch via `aiohttp`
  - `_fetch_rss_feed_async()` — per-feed async parser
  - `_format_news_response_plain()` — numbered headline, 1–2 sentence info, source/timestamp, `Read more: URL`
  - `_format_news_response_voice()` — natural Kokoro TTS format
  - `_format_news_response_html()` — styled HTML with clickable links
  - **No `_news_cache` — deleted entirely**

- **`tools/router_py/execution_engine.py`** — NEWS route handling:
  - `_fetch_news_evidence()` — calls async `NewsProvider.fetch_news()` directly
  - NEWS route `response_text` — plain text from `evidence["context"]`
  - `voice_text` — built from article metadata, passed via `metadata`
  - `news_partial` and `news_errors` in metadata for UI

- **`ui-v9/app/panels/conversation_panel.py`** — Link rendering:
  - `self._history` changed from `QTextEdit` → `QTextBrowser`
  - `setOpenExternalLinks(True)` — clicks open default browser
  - `_auto_link_urls()` — wraps `http://...` in `<a href="...">` tags
  - Route path labels: `News (RSS)`, `Time lookup`
  - Route confidence suppressed when `0.0`

- **`ui-v9/app/services/runtime_bridge.py`** — History suppression:
  - `_write_history_entry()` skipped when `result.route == "NEWS"`

### Rigid Rules (Enforced by Tests)

1. **No cache** — `_news_cache` dict does not exist. Every news query fetches all 14 feeds fresh.
2. **No history** — NEWS results display in Latest Answer but never write to `request_history.jsonl`.
3. **No LLM** — NEWS route bypasses Ollama entirely. Raw RSS content only.
4. **Direct sources** — Every article carries its original source name and URL.

**Result:** Every time query returned "Time tool not found" → "Sorry, I couldn't find the time..."

## 3. HMI Crash Fix

**Root cause:** When switching `self._history` from `QTextEdit` to `QTextBrowser`, the `QTextEdit` import was accidentally removed from the import list. But `self._draft = QTextEdit()` (operator input box) still needed it.

**Result:** `NameError: name 'QTextEdit' is not defined` on HMI startup.

**Fix:** Added `QTextEdit` back to imports alongside `QTextBrowser`.
