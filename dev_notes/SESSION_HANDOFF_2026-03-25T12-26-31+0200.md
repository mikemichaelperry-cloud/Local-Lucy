## Root Cause Found
1. HMI desktop launcher runs non-interactive login shell: `bash -lc` from `~/Desktop/Local Lucy HMI.desktop`.
2. `OPENAI_API_KEY` was exported only in `~/.bashrc`, after the interactive-shell guard (`case $- ... return`), so non-interactive desktop launch paths never saw it.
3. Runtime path (`ui -> runtime_bridge.py -> runtime_request.py -> unverified_context_openai.py`) correctly preserved inherited env and failed explicitly when key was missing.
