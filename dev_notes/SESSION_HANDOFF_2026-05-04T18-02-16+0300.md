## What Was Done

### Bug Fix: Transistor Query → News Headlines
**File:** `tools/unverified_context_trusted.py`  
**Line:** 138 (in `_is_category_supported()`)

**Problem:** When asking "Is the transistor 2N3055 still in common usage today?", the classifier correctly routed to `LOCAL`. However, the local model refused the query as time-sensitive (because of the word "today"). The execution engine's provisional fallback then called the augmented provider chain. The `trusted` provider was tried first because `_is_category_specific_query()` returned `True` for any query with `intent_family == "current_evidence"`. The trusted provider's `_is_category_supported()` then matched `intent_family == "current_evidence"` → treated it as a news query → fetched world news RSS headlines → returned them as the answer.

**Fix:** Removed the overly broad `intent_family == "current_evidence"` condition from the news-detection branch in `_is_category_supported()`. Now the trusted provider only handles queries that explicitly contain news keywords (`news`, `headline`, `headlines`, `breaking`, `latest news`, `world news`), or fall into medical/finance categories.

**Before:**
```python
if intent_family == "current_evidence" or any(kw in q_lower for kw in news_keywords):
```

**After:**
```python
if any(kw in q_lower for kw in news_keywords):
```

## Architecture Notes

### Split-Brain Reminder
- **UI code** loads from workspace (`ui-v9/app/`)
- **Backend code** loads from snapshot (`snapshots/opt-experimental-v9-dev/tools/`)
- Backend files must be **manually synced** to the snapshot after editing workspace copies

### Route Flow for "Is the 2N3055 still in common usage today?"
1. **Classifier** → `LOCAL` (`intent_family=current_evidence`, `category=electronics`)
2. **Execution engine** → `_execute_provisional_route()` → `_execute_bypass_route()`
3. **Local model** → `_is_time_sensitive()` matches "today" → returns "This requires evidence mode"
4. **Sufficiency check** → `_is_local_response_sufficient()` sees "requires evidence mode" → `False`
5. **Fallback** → `_call_augmented_provider()` → provider chain: `[trusted, wikipedia, openai, kimi]`
6. **Trusted provider** → `_is_category_supported()` → **now returns `(None, None)`** because no news/medical/finance keywords present → `not_applicable`
7. **Wikipedia provider** → fetches article about 2N3055 transistor → local model synthesizes answer
8. **Result** → AUGMENTED answer about the 2N3055, not news headlines

### Policy `fallback_only` (default)
- Classifier returns `LOCAL` for `current_evidence`
- Engine may override via provisional execution if local response is insufficient
- This is intentional — balances speed vs. quality
