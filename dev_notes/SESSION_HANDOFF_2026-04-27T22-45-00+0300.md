## 3. News UI Bugfixes ✅

### Bug 1: HTML stripped from news responses
**Root cause:** `conversation_panel.py` ran all responses through `_operator_response_text()`, which strips HTML line-by-line.

**Fix:** For non-failure responses with HTML content, embed the raw `response_text` directly:
```python
raw_response = self._entry_text(entry, "response_text") or ""
if not has_failure and self._is_html_content(raw_response):
    return self._build_html_response(..., f"<b>Latest Answer</b><br>{raw_response}")
```

### Bug 2: Article descriptions missing
**Root cause:** ElementTree Elements containing CDATA text (no child elements) are **falsy** in Python boolean context. The `or` chain in `_fetch_rss_feed()` silently discarded every description:
```python
desc_elem = item.find("description") or item.find("summary")
#        ^-- Element with text but no children = falsy
```

**Fix:** Explicit `is None` checks instead of `or`:
```python
desc_elem = item.find("description")
if desc_elem is None:
    desc_elem = item.find("summary")
```

### Result
Text-mode news now shows:
- **Headline** + **Source/time**
- **Description** (article summary)
- **"Read more"** clickable link → opens in system browser

Voice mode still gets headlines-only (by design).
