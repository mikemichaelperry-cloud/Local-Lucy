## What Was Done in This Session

### 1. Fix Background Learner — Deduplication Persistence Bug
**File:** `models/router/background_learner.py`

- **Root cause:** `learn_once()` only saved the index when `added > 0` (new examples). When user feedback *corrected* an existing query's route (same query, different label), `added == 0` so the corrected index was never persisted.
- **Fix:** Added `index_changed = added > 0 or all_examples != examples` so any mutation triggers save + embedding rebuild.

**Demonstrated:** Query *"What day was I born?"* misrouted to `TIME` → feedback submitted → learned → re-queried → correctly routed to `LOCAL`.

### 2. Fix Background Learner — Atomic Save Cross-Device Link
**File:** `models/router/background_learner.py`

- **Root cause:** `save_index()` used `tempfile.NamedTemporaryFile(dir=ROUTER_DIR)` then `os.replace(tmp.name, INDEX_PATH)`. When tests patched `INDEX_PATH` to a different filesystem (e.g., `/tmp`), `os.replace()` failed with `OSError: [Errno 18] Invalid cross-device link`.
- **Fix:** `tmp_dir = str(Path(INDEX_PATH).parent)` — temp file is created in the same directory as the destination, guaranteeing atomic rename works.

### 3. Improve Feedback Parser — Explicit Route Correction Patterns
**File:** `tools/router_py/feedback_parser.py`

- **Root cause:** Phrases like *"Incorrect, the correct routing is LOCAL"* were detected as generic `ANSWER_NEGATIVE` (from the word "Incorrect"), but the explicit "LOCAL" was ignored because no route-correction pattern matched the sentence structure.
- **Fix:** Added 3 new regex patterns to `ROUTE_CORRECTION_PATTERNS`:
  - `correct\s+routing\s+(?:is|was)\s+ROUTE_NAME`
  - `correct\s+answer\s+(?:is|was)\s+ROUTE_NAME`
  - `(?:it's|it is)\s+ROUTE_NAME\b`

Now explicit corrections are captured as `ROUTE_CORRECTION` with `confidence=1.0` instead of relying on heuristic inference (capped at `0.7`).

### 4. Create Manual E2E Feedback Loop Demo
**New file:** `manual_feedback_loop_demo.py`

- End-to-end script that:
  1. Backs up production router files
  2. Finds (or forces) a misrouted query
  3. Shows pre-learn route
  4. Submits corrective feedback
  5. Triggers `background_learner.learn_once()`
  6. Re-queries and verifies corrected route
  7. Restores production files
- **Result:** Successfully demonstrated live learning — *"What day was I born?"* flipped from `TIME` (0.992) to `LOCAL` (0.995).
