## Executive Summary

This session completed the NEWS route fixes started in the previous session. The **root problems were:**

1. **Voice cut off early** — TTS subprocess had a hard 60s timeout, but 10 news articles ≈ 100s of audio + synthesis overhead
2. **HTML display dropped the request header** — `_format_operator_entry` returned raw HTML answer body without "Latest Request" section

## Changes Made

### 1. TTS Subprocess Timeout (`runtime_bridge.py`)

**File:** `~/lucy-v8/ui-v8/app/services/runtime_bridge.py`

**Before:**
```python
subprocess.run(
    ["python3", str(self.voice_tool_path), "speak", "--text", text],
    timeout=60,
    ...
)
```

**After:**
```python
speak_timeout = max(60, len(text) // 8 + 30)
speak_timeout = min(speak_timeout, 300)
subprocess.run(
    ["python3", str(self.voice_tool_path), "speak", "--text", text],
    timeout=speak_timeout,
    ...
)
```

Rationale: ~8 chars/second spoken rate + 30s synthesis headroom. A 1200-char news summary gets 180s instead of 60s.

### 2. HTML Display with Request Header (`conversation_panel.py`)

**File:** `~/lucy-v8/ui-v8/app/panels/conversation_panel.py`

**Before:** When HTML was detected, `_format_operator_entry` returned ONLY `result_text` (raw answer HTML), losing the "Latest Request" header entirely.

**After:** Builds a proper HTML document:
```python
if self._looks_like_html(result_text):
    escaped_request = html.escape(request_text)
    body_match = re.search(r'<body[^>]*>(.*?)</body>', result_text, re.IGNORECASE | re.DOTALL)
    answer_content = body_match.group(1).strip() if body_match else result_text
    answer_content = re.sub(r'</?html[^>]*>', '', answer_content, flags=re.IGNORECASE)
    return (
        '<html><body style="font-family: sans-serif; font-size: 13px; color: #d8e0e6;">'
        f'<p style="margin: 4px 0;"><b>Latest Request</b><br>{escaped_request}</p>'
        '<hr style="border: none; border-top: 1px solid #3a4a5a; margin: 8px 0;">'
        f'<p style="margin: 4px 0;"><b>{result_title}</b></p>'
        f'{answer_content}'
        '</body></html>'
    )
```

### 3. Voice Text: All Articles (`news_provider.py` + `execution_engine.py`)

**Files:**
- `~/lucy-v8/tools/router_py/news_provider.py`
- `~/lucy-v8/tools/router_py/execution_engine.py`

Removed `articles[:6]` → `articles` in both voice text assembly locations. Display already showed up to 10 articles; voice was arbitrarily limited to 6.

### 4. Live Entry Metadata (`main_window.py`)

**File:** `~/lucy-v8/ui-v8/app/main_window.py`

Live entries built by `_build_live_entry_from_payload` now carry `"metadata": payload.get("metadata")` so `voice_text` is available for TTS replay.

### 5. Failed NEWS Display (`main_window.py`)

Failed NEWS requests (status != "completed") now also generate live entries via the same `_build_live_entry_from_payload` path, so errors are visible in the HMI.

### 6. Ollama Lock Deadlock Fix (`local_answer.py`)

**File:** `~/lucy-v8/tools/router_py/local_answer.py` (synced to snapshot)

**Before:** `threading.Lock()` acquired with `with _ollama_call_lock:` across `await` calls — blocks event loop, creates deadlock risk under concurrent requests.

**After:**
```python
async def _acquire_ollama_call_lock() -> None:
    await asyncio.to_thread(_ollama_call_lock.acquire)

await _acquire_ollama_call_lock()
try:
    async with session.post(...) as response:
        ...
finally:
    _ollama_call_lock.release()
```

### 7. NEWS Exempt from Time-Sensitive Refusal (`local_answer.py`)

NEWS queries were incorrectly routed through time-sensitive check and refused. Added `"NEWS"` to the exemption set alongside `"AUGMENTED"`.

### 8. Missing `urllib.parse` Import (`news_provider.py`)

NewsAPI path uses `urllib.parse.quote()` but the import was missing. Added `import urllib.parse`.

### 9. aiohttp Hard Dependency (`news_provider.py`)

Removed the broken `try/except ImportError` fallback for aiohttp. The module is listed in `requirements.txt` and the fallback code path would crash on use anyway.
