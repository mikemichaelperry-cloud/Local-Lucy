## 2. GPUâ†’CPU Fallback âś…

### Implementation
Both sync (`runtime_voice.py`) and async (`router_py/voice_tool.py`) pipelines now:
1. Attempt GPU first (whisper default)
2. If exit code â‰  0 and stderr contains `cuda`, `cublas`, `gpu`, `oom`, `out of memory`
3. Retry with `--no-gpu`
4. Report `fallback_used=True` + `fallback_reason` in `voice_runtime.json`

## 7. Architectural Thinking (for next session)

### Persistent whisper worker
The biggest remaining latency win is a `whisper_session_worker.py` (like kokoro TTS already uses) that loads the model once into GPU memory and accepts socket requests. Would cut STT from ~600 ms â†’ ~150 ms for short audio.

### Query pre-filter
Before `classify_intent` + `ExecutionEngine`, a fast keyword/regex layer could shortcut common voice commands ("lights on", "what time") directly to local tools, skipping LLM latency entirely.

### File splitting
`runtime_voice.py` (~2400 lines) mixes 6 responsibilities: CLI, backend detection, recording, STT, TTS, submission, state management. Logical split:
```
tools/voice/backend.py   # detection
tools/voice/stt.py       # whisper/vosk + fallback
tools/voice/tts.py       # kokoro/piper + worker
tools/voice/recorder.py  # arecord/pw-record
tools/voice/state.py     # voice_runtime.json
tools/voice/submit.py    # bridge to Lucy router
tools/voice/cli.py       # argparse + main
```
