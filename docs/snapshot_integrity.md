# Snapshot Integrity

`SHA256SUMS.clean` is the clean-file checksum manifest for the active Local Lucy snapshot.

## What It Covers

The manifest is generated by `tools/sha_manifest.sh` and tracks files from:

- `config/`
- `tools/`
- `runtime/`
- `lucy_chat.sh`

Because it walks those trees broadly, it also covers checked-in test files, fixtures, and checked-in runtime source bundles under those paths unless they are explicitly excluded by the script.

The intent is to cover the snapshot's executable source, config, and runtime logic that should stay stable for a frozen hygiene baseline.

## What It Excludes

It intentionally excludes mutable or bulky paths, including:

- `docs/`
- `dev_notes/`
- `logs/`
- `out/`
- `build/`
- `vendor/`
- `.git/`, `.github/`, `.devops/`, `.idea/`
- `__pycache__/`, `*.pyc`, `*.tmp`, `*.bak*`
- generated or local-state files such as:
  - `runtime/state.json`
  - `runtime/Modelfile.local-lucy-mem.generated`
  - temporary trust/url-map test files
- large voice assets and local environments such as:
  - `runtime/voice/piper-venv/`
  - `runtime/voice/models/`
  - `runtime/voice/whisper.cpp/models/`

So a checksum pass is a source/config/runtime integrity check, not a byte-for-byte archive of the entire snapshot directory.

## When To Regenerate

Run `tools/sha_manifest.sh regen` when you intentionally change a tracked file and want that new state to become the frozen clean baseline.

Typical cases:

- edits under `config/`
- edits under `tools/`
- edits under tracked `runtime/` source
- edits to `lucy_chat.sh`

Do not regenerate it just to silence an unexpected mismatch. First decide whether the tracked change is intentional and should become part of the frozen baseline.

## Self-Contained Snapshot

In Local Lucy, a self-contained snapshot means the snapshot can be launched, validated, and reasoned about from its own root without depending on edits in another snapshot root.

Practically, that means:

- launchers for that snapshot resolve into that snapshot's root
- code, config, tools, and runtime logic used by the launcher live in that snapshot
- validation and handoff state are tied to that single active snapshot root
- frozen or stable sibling snapshots are not edited as part of the same pass

Self-contained does not mean every large optional asset is checksummed inside `SHA256SUMS.clean`; it means the snapshot owns its executable behavior and integrity boundary.
